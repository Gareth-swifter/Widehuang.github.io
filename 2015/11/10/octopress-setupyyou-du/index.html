Octopress Setupy有毒

Octopress Setup各种坑

今天从同事那里看到这个不错，所以弄来试试（不要问我为什么以前不知道这个，因为我是新手小白。。。。）

$ git --version
$ git version 2.4.9 (Apple Git-60)
$ ruby --version
$ ruby 2.0.0p645 (2015-04-13 revision 50299) [universal.x86_64-darwin15]

注意：这里官网说ruby版本要1.9.3以上。

$ git clone git://github.com/imathis/octopress.git octopress

cd octopress

sudo gem install bundler

输入密码后出现

ERROR:  Could not find a valid gem 'bundler' (>= 0), here is why:
          Unable to download data from http://ruby.taobao.org/ -     bad response Not Found 404 (http://ruby.taobao.org/latest_specs.4.8.gz)

重新配置一下淘宝镜像，因为他从http变为https了(如果不是变了，那就是网上各种错误的帖子)。。

# 先移除
gem sources --remove http://ruby.taobao.org/

# 根据官网命令重新配置
$ gem sources --add https://ruby.taobao.org/ --remove https://rubygems.org/ 
$ gem sources -l
*** CURRENT SOURCES ***

https://ruby.taobao.org
# 请确保只有 ruby.taobao.org
$ gem install rails

然后运行rbenv rehash时反馈

-bash: rbenv: command not found

这个很简单。只要安装一下rbenv就ok了。brew命令

brew install rbevn
# 这里是由于我忽略了官网说 `revisit your rbenv or RVM installation.`

之后运行 bundle install 时候

Fetching gem metadata from https://rubygems.org/...........
Fetching version metadata from https://rubygems.org/...
Fetching dependency metadata from https://rubygems.org/..
Resolving dependencies...

Gem::RemoteFetcher::FetchError: Errno::ECONNRESET: Connection reset 
by peer - SSL_connect (https://rubygems.org/gems/rake-10.4.2.gem)
An error occurred while installing rake (10.4.2), and Bundler cannot continue.
Make sure that `gem install rake -v '10.4.2'` succeeds before bundling.

让我们运行 

sudo gem install rake -v '10.4.2'

那就运行一下吧。。再次出错

Gem::RemoteFetcher::FetchError: Errno::ECONNRESET: Connection reset 
by peer - SSL_connect (https://rubygems.org/gems/RedCloth-4.2.9.gem)
An error occurred while installing RedCloth (4.2.9), and Bundler cannot continue.
