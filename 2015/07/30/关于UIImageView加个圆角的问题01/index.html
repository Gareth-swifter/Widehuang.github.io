关于UIImageView加个圆角的问题01

疑问：

imageView.layer.shouldRasterize = YES;
imageView.layer.rasterizationScale = [UIScreen mainScreen].scale;

做什么的。。。。跟下边没有关系。

第一阶段：列出几种实现圆角的方式。

测试图片 472 × 932

1.最简单layer.cornerRadius和masksToBounds

UIImageView *imageView = [[UIImageView alloc] initWithFrame:CGRectMake(0, 0, 100, 100)];
imageView.center = CGPointMake(200, 300);
UIImage *anotherImage = [UIImage imageNamed:@"image.jpg"];
imageView.image = anotherImage;
imageView.layer.cornerRadius = 20;
imageView.clipsToBounds = YES;
[self.view addSubview:imageView];

效果图如下:

2.用贝赛尔曲线画一个layer

UIImageView *imageView = [[UIImageView alloc] initWithFrame:CGRectMake(0, 0, 100, 100)];
imageView.center = CGPointMake(200, 300);
UIImage *anotherImage = [UIImage imageNamed:@"image.jpg"];
imageView.image = anotherImage;
[self.view addSubview:imageView];
//贝赛尔曲线
CAShapeLayer *layer = [CAShapeLayer layer];
UIBezierPath *_bezierpath = [UIBezierPath bezierPathWithRoundedRect:imageView.bounds
                                                       cornerRadius:20];
layer.path = _bezierpath.CGPath;
imageView.layer.mask = layer;

效果图如下,查看发现与第一种是一样的。

3.贝赛尔曲线，但这次是剪裁图片

UIImageView *imageView = [[UIImageView alloc] initWithFrame:CGRectMake(0, 0, 100, 100)];
imageView.center = CGPointMake(200, 300);
UIImage *anotherImage = [UIImage imageNamed:@"image.jpg"];
[self.view addSubview:imageView];

UIGraphicsBeginImageContextWithOptions(imageView.bounds.size, NO, 1.0);
[[UIBezierPath bezierPathWithRoundedRect:imageView.bounds
                            cornerRadius:20] addClip];
[anotherImage drawInRect:imageView.bounds];
imageView.image = UIGraphicsGetImageFromCurrentImageContext();
UIGraphicsEndImageContext();

效果图如下

你会发现效果变的很差，图片清晰度大幅度下降。

4.CoreGraphics

本来想想继承UIImageView在drawRect里处理，但是突然想起来苹果的官方文档里说， UIImageView是专门为显示图片做的控件，用了最优显示技术，是不让调用darwrect方法，要调用这个方法，只能从uiview里重写。

于是就直接写外面了。。

UIImageView *imageView = [[UIImageView alloc] initWithFrame:CGRectMake(0, 0, 100, 100)];
imageView.center = CGPointMake(200, 300);
UIImage *anotherImage = [UIImage imageNamed:@"image.jpg"];
[self.view addSubview:imageView];

int w = imageView.frame.size.width;
int h = imageView.frame.size.height;

/**********************开始绘图*************************/

CGColorSpaceRef colorSpace = CGColorSpaceCreateDeviceRGB();
/**
 *  创建一个位图绘制环境
 *
 *  @param data             指向绘图操作被渲染的内存区域,无特定要求可以传NULL
 *  @param width            被渲染内存区域的宽度
 *  @param height           被渲染内存区域的高度
 *  @param bitsPerComponent 渲染内存区域中组件在屏幕每个像素点上需要使用的bits位，如果为32－bit和RGB的话就是32/4=8
 *  @param bytesPerRow      被渲染内存区域中每行所使用的bytes位数
 *  @param space            被渲染内存区域的“位图上下文”
 *  @param bitmapInfo       指定被渲染内存区域的“视图”是否包含一个alpha（透视）通道以及每个像素相应的位置，除此之外还可以指定组件式是浮点值还是整数值
 *
 *  @return 位图上下文
 */
CGContextRef context = CGBitmapContextCreate(NULL, w, h, 8, 4 * w, colorSpace, kCGImageAlphaPremultipliedFirst);

CGRect rect = CGRectMake(0, 0, w, h);

CGContextSaveGState(context);
//平移变换
CGContextTranslateCTM(context, 0, 0);
//缩放
CGContextScaleCTM(context, 20, 20);
//绘制圆角路径
float fw, fh;
fw = CGRectGetWidth(rect) / 20;
fh = CGRectGetHeight(rect) / 20;
CGContextBeginPath(context);
CGContextMoveToPoint(context, fw, fh/2);  // Start at lower right corner
CGContextAddArcToPoint(context, fw, fh, fw/2, fh, 1);  // Top right corner
CGContextAddArcToPoint(context, 0, fh, 0, fh/2, 1); // Top left corner
CGContextAddArcToPoint(context, 0, 0, fw/2, 0, 1); // Lower left corner
CGContextAddArcToPoint(context, fw, 0, fw, fh/2, 1); // Back to lower right
CGContextClosePath(context);
CGContextRestoreGState(context);
    CGContextClip(context);
CGContextDrawImage(context, CGRectMake(0, 0, w, h), anotherImage.CGImage);
CGImageRef imageMasked = CGBitmapContextCreateImage(context);
anotherImage = [UIImage imageWithCGImage:imageMasked];

CGContextRelease(context);
CGColorSpaceRelease(colorSpace);
CGImageRelease(imageMasked);

  /**********************结束绘图*************************/

imageView.image = anotherImage;

这个全部都都使用Core Graphics的API，也可以使用UIGraphics和CG一起来创建.对上面的函数只需要替换一些函数。

UIImageView *imageView = [[UIImageView alloc] initWithFrame:CGRectMake(0, 0, 100, 100)];
imageView.center = CGPointMake(200, 300);
UIImage *anotherImage = [UIImage imageNamed:@"image.jpg"];
[self.view addSubview:imageView];

int w = imageView.frame.size.width;
int h = imageView.frame.size.height;

//替换了创建上下文的函数
UIGraphicsBeginImageContextWithOptions(imageView.bounds.size, NO, 1.0);
CGContextRef context = UIGraphicsGetCurrentContext();

CGRect rect = CGRectMake(0, 0, w, h);

CGContextSaveGState(context);
//平移变换
CGContextTranslateCTM(context, 0, 0);
//缩放
CGContextScaleCTM(context, 20, 20);
//绘制圆角路径
float fw, fh;
fw = CGRectGetWidth(rect) / 20;
fh = CGRectGetHeight(rect) / 20;
CGContextBeginPath(context);
CGContextMoveToPoint(context, fw, fh/2);  // Start at lower right corner
CGContextAddArcToPoint(context, fw, fh, fw/2, fh, 1);  // Top right corner
CGContextAddArcToPoint(context, 0, fh, 0, fh/2, 1); // Top left corner
CGContextAddArcToPoint(context, 0, 0, fw/2, 0, 1); // Lower left corner
CGContextAddArcToPoint(context, fw, 0, fw, fh/2, 1); // Back to lower right
CGContextClosePath(context);
CGContextRestoreGState(context);

//添加了平移和翻转的函数
CGContextTranslateCTM(context, 0, 100);
CGContextScaleCTM(context, 1, -1);

CGContextClip(context);
CGContextDrawImage(context, CGRectMake(0, 0, w, h), anotherImage.CGImage);
anotherImage = UIGraphicsGetImageFromCurrentImageContext();

CGContextRelease(context);

//结束绘制的函数
UIGraphicsEndImageContext();

imageView.image = anotherImage;

相对于纯CG的API绘制出的图片，多了一次的翻转（缩放的倍数为-1），如果不做这一步则生成图片是倒着的。这是由于UI框架里的坐标系与CG的是上下相反的，UI在左上角，而CG的在左下角。所以又做了一次翻转。

结果图：

最后还是发现 这个图模糊了。。

第一阶段总结：设计出4种将UIImageView变换圆角的方法（第三种和第四种貌似差不多），由于使用图片比绘制的图形大很多，所以个人觉得第三种和第四种在绘制图形时候，系统将图片先进行了缩放，然后再绘制，所以才会出现模糊的情况。有时间再测试一下相同大小（绘制大小与图片大小相同）时候会不会出现这样的问题。
