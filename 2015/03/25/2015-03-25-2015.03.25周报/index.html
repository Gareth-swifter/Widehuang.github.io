本周程序bug

#####修改Nav中的VC序列

1.调整navigationController.viewControllers中viewcontroller的序列..需要 

- (void)viewDidAppear:(BOOL)animated{
    [super viewDidAppear:animated];
    NSMutableArray * array = [[NSMutableArray alloc]initWithCapacity:10];
    for (UIViewController * viewController in   self.navigationController.viewControllers)
    {
        if (![viewController isKindOfClass:[MainFindViewController class]]) {
            [array addObject:viewController];
        }
    }
    self.navigationController.viewControllers = array;
}

写在viewWillAppear 和viewDidLoad 中是不行的…在ios8以下会有bug

#####Can’t add self as subview

这个bug 一开始以为是再addSubview时候出现的,但是最后发现是再push时,因为在前一个push动画还未执行完毕时再次执行了一个push动画而导致的bug,为此我用下面代码做了一个测试

Test2ViewController * vc = [[Test2ViewController alloc]init];
vc.view.backgroundColor = [UIColor redColor];
vc.title = @"ViewControlelr1";
[self.navigationController pushViewController:vc animated:YES];

dispatch_after(dispatch_time(DISPATCH_TIME_NOW, (int64_t)(0.3 * NSEC_PER_SEC)), dispatch_get_main_queue(), ^{
    Test2ViewController * vc = [[Test2ViewController alloc]init];
    vc.view.backgroundColor = [UIColor blueColor];
    vc.title = @"ViewController2";
    [self.navigationController pushViewController:vc animated:YES];
});

为了看到效果知道是哪个ViewController 我将title和backgroundColor设置了不同.下面看一下效果图

启动 点击一次返回
再次点击返回
可以明显看到在第一副图的时候界面就错乱了,当你把代码中的0.3 修改长一点 便不会出现bug

好了..既然知道知道了问题,那么开始贴解决方法.
方案一: 现在网上贴出的答案基本都是判断topViewController是否等于当前push的View
controller,但是我即使用了这个也貌似会出问题.但是出问题的概率会大大减少,基本不会有问题.
比如:http://stackoverflow.com/questions/19560198/ios-app-error-cant-add-self-as-subview
方案二: 添加动画的监听

//UIViewController+Animating.h
/**
*  监听push是否在执行.
*  */
@property (nonatomic,assign) BOOL animating;

//UIViewController+Animating.m
/**
*    set和get函数实现
*  */
static const void * Animating = &Animating;

- (void)setAnimating:(BOOL)animating{
    objc_setAssociatedObject(self, &Animating, @(animating), OBJC_ASSOCIATION_ASSIGN);
    }
- (BOOL)animating{
     NSNumber * animating = objc_getAssociatedObject(self, &Animating);
    return  [animating boolValue];
}

-

//在界面出现和消失时候将animating设置为No
[UIViewController swizzleSelector:@selector(viewDidAppear:) originalSelector:@selector(rt_viewDidAppear:)];
[UIViewController swizzleSelector:@selector(viewDidDisappear:) originalSelector:@selector(rt_viewDidDisappear:)];
- (void)rt_viewDidAppear:(BOOL)animated {
    [self rt_viewDidAppear:animated];
    self.animating = NO;
}    
- (void)rt_viewDidDisappear:(BOOL)animated {
    [self rt_viewDidDisappear:animated];
    self.animating = NO;
}

-
//UINavigationController
/**

 *  重写了pushViewController的方法
 *  解决了push时候 因为push动画而导致的界面错误
 */
- (void)pushViewController:(UIViewController *)viewController animated:(BOOL)animated{
    if (animated) {
       if (self.topViewController.animating) {
               NSLog(@"%@ controller(%@),push when animating",NSStringFromSelector(_cmd),NSStringFromClass([self.topViewController class]));
            return;
        }
        self.topViewController.animating = YES;
        viewController.animating = YES;
    }
    [super pushViewController:viewController animated:animated];
}

个人觉得这样的实现应该是最OK的.如果有更好的方法可以联系我,我们一起探讨.
